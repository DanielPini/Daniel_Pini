Earthly Futures Calendar Integration

Events & booking calendar with recurrence handling, admin interface, and performance-conscious loading strategy.

Goal

Enable staff to schedule recurring events and users to browse quickly without overload.

Features

  • Recurrence rule parsing (weekly, monthly patterns).
  • Admin UI for create/update/delete events.
  • Lazy month data fetching & caching layer.

Architecture

events.json / events-api
recurrence parser → expand instances
calendar grid → virtualized weeks
admin panel → form + validation

Challenges & Solutions

  • Complex recurrence edge cases → normalized rule schema.
  • Initial load clutter → deferred future months.
  • Admin error handling → inline validation & autosave drafts.

Performance

Reduced initial event payload by loading only current + next month; cached expansions for repeat visits.

Outcome

Simplified scheduling workflow and improved user navigation speed.
