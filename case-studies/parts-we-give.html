The Parts We Give (Web Game Engine)

Immersive prelude to a modern opera: bespoke JavaScript engine, audio-reactive pacing, and lightweight entity/state system.

Goal

Create a web-playable intro with synchronized audio cues, fluid scene transitions, and low setup friction for users.

Engine Design

// Core Loop
requestAnimationFrame(loop)
processInput()
updateEntities(dt)
dispatchAudioCues()
render()
  • Entity registry with component-like data objects.
  • Timed audio triggers via Howler and timeline offsets.
  • State manager for scene transitions (Intro → Exploration → Finale).

Audio Integration

Mapped critical musical phrases to spawn, lighting, and pacing adjustments using Howler events and timestamp arrays.

Challenges & Solutions

  • Desync risk → centralized timebase using performance.now().
  • Asset load delays → prefetch + progressive scene activation.
  • Input latency → simplified collision & no heavy physics.

Performance

Metric Target Achieved
Frame Rate (avg) 60 FPS 59–60
Initial Load (network) < 2.5 MB 2.1 MB
Input-to-effect latency < 50 ms 32 ms

Outcome

Delivered a polished interactive intro with stable performance and synchronized audio drama cues.
